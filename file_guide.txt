# 📘 Alarm プロジェクト ファイル構成ガイド

（「どのファイルが、何をするか」一覧）

---

## 🟢 起動・エントリーポイント系

### 🔹 `start.py`

**アプリの起点（main）**

* GUI / CUI を切り替えて起動する入口
* 最初に実行されるファイル
* ここには「判断」だけを書く（ロジックは書かない）

👉 *人間でいう「玄関」*

---

### 🔹 `__init__.py`

**プロジェクト全体の初期化**

* `BASE_DIR` などの共通パス定義
* モジュールとして alarm を成立させるための土台

👉 *地面・基礎工事*

---

## 🟢 データモデル・設計の中核

### 🔹 `alarm_model.py`

**データの定義（設計図）**

* `AlarmInternal`
* `AlarmJson`
* `AlarmStateJson`

などの **dataclass 定義**

* 「アラームとは何か？」を定義する唯一の場所
* ロジックは書かない

👉 *設計図・型・仕様書*

---

### 🔹 `constants.py`

**全体で共有する定数辞書**

* repeat 種類
* 曜日表示
* GUI 用ラベル
* JSON キー定義

👉 *用語集・共通ルールブック*

---

## 🟢 アラームの中枢ロジック

### 🔹 `alarm_manager.py`

**アラーム全体の司令塔**

* アラームの追加・削除・更新
* 有効 / 無効管理
* Scheduler / Storage / Player をつなぐ

👉 *オーケストラの指揮者*

---

### 🔹 `alarm_scheduler.py`

**「次にいつ鳴るか」を計算する**

* 入力：`AlarmInternal` + `now`

* 出力：次回の `datetime`

* repeat（none / daily / weekly / monthly / custom）ごとに計算

* **未来予測エンジン**

👉 *日時計算専門の頭脳*

---

### 🔹 `alarm_repeat_rules.py`

**「今鳴らしていいか？」を判定する**

* Scheduler が「予定」を出すのに対し、

* こちらは「今この瞬間、条件を満たすか？」を判定

* skip_holiday

* interval_weeks

* last_fired 判定など

👉 *ルールブック・審判*

---

## 🟢 永続化・入出力

### 🔹 `alarm_storage.py`

**保存と復元の責務**

* alarms.json

* standby.json

* backup 管理

* JSON ⇄ dataclass の橋渡しは Loader に任せる

👉 *倉庫・金庫*

---

### 🔹 `alarm_loader.py`

**形式変換専門**

* dict → AlarmJson → AlarmInternal
* datetime / str の変換
* 保存形式と内部形式の分離

👉 *通訳・変換機*

---

### 🔹 `external_loader.py`

**外部データ取り込み**

* 将来拡張用（CSV / 外部設定など）
* コアロジックとは独立

👉 *外部インターフェース*

---

## 🟢 再生・UI 系

### 🔹 `alarm_player.py`

**音を鳴らすだけ**

* pygame 再生
* 停止
* duration 制御

👉 *スピーカー係（判断しない）*

---

### 🔹 `gui.py`

**GUI 全体の制御**

* Tkinter（将来 PySide）
* ボタン・ダイアログ・一覧表示
* AlarmManager を呼ぶだけ
(重要)
    # --------------------------------------------
    # 🔹 カスタム繰り返し設定ウインドウ（ロジック要注意）
    # --------------------------------------------
    # 【設計ルール】
    # カスタム繰り返しは、
    # ・月内ルール（第n週 + 曜日）
    # ・時系列ルール（n週おき + 曜日）
    # のいずれか一方のみを選択する。
    #
    # 両者は参照基準が異なるため、
    # 同時指定は意味を持たず、UIレベルで禁止する。

👉 *操作パネル*

---

### 🔹 `cui.py` / `sample_cui.py`

**CUI 操作用**

* デバッグ・テスト用
* GUI に依存しない操作確認

👉 *裏口・整備用端末*

---

### 🔹 `mini_calendar.py`

**日付選択専用 UI**

* カレンダー表示
* 日付入力補助

👉 *カレンダー部品*

---

### 🔹 `json_editor.py`

**JSON 直接編集ツール**

* alarms.json / standby.json を人間が触るための補助
* デバッグ・緊急修正用

👉 *整備士用ツール*

---

## 🟢 ユーティリティ系

### 🔹 `utils.py`

**汎用関数置き場**

* validate_date
* validate_time
* 小さな共通処理

👉 *工具箱*

---

### 🔹 `date_rule_util.py`

**日付計算の補助ロジック**

* normalize_base_date
* 週差・基準日補正

👉 *日付計算の補助脳*

---

### 🔹 `path_utils.py`

**パス操作専門**

* OS 依存を吸収
* 保存先の正規化

👉 *案内係*

---

## 🟢 テスト

### 🔹 `alarm_test_full_condition.py`

**Scheduler の仕様確認テスト**

* repeat 全パターン検証
* expected = 仕様そのもの

👉 *品質保証・仕様書*

---

## 🎯 全体の役割分担（超重要まとめ）

| 種類 | ファイル                                 |
| -- | ------------------------------------ |
| 定義 | alarm_model / constants              |
| 判断 | alarm_scheduler / alarm_repeat_rules |
| 管理 | alarm_manager                        |
| 保存 | alarm_storage / alarm_loader         |
| UI | gui / cui / mini_calendar            |
| 補助 | utils / date_rule_util               |
| 起動 | start                                |
